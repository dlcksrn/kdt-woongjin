# 데이터 분석 프로젝트 목표 (모니터링 관점)

본 문서는 `seoul-subway-monitor` 프로젝트의 데이터 분석 목표와 방법론을 기술합니다.

## 1. 배차 간격 정기성 분석 (Interval Regularity)
- **목표**: 특정 역에 도착하는 열차들 사이의 시간 간격을 계산하여 배차가 밀리거나(Bunching) 너무 벌어지는 구간을 탐지합니다.
- **방법**:
    - 동일 역, 동일 노선, 동일 방향의 열차 도착 시간(`created_at` 또는 `last_rec_time`)을 기준으로 이전 열차와의 시간 차이를 계산.
    - 표준 배차 간격과 비교하여 이상치 탐지.

## 2. 지연 발생구간 탐지 (Delay Hotspots)
- **목표**: 역별 체류 시간(`train_status` 1(도착) -> 2(출발) 변화 시간)을 측정하여 평시보다 오래 멈춰있는 역을 실시간 시각화합니다.
- **방법**:
    - `train_number` 기준으로 로그를 추적하여 각 역에서의 체류 시간 계산.
    - 특정 역에서 평균 체류 시간보다 현저히 긴 경우 경고.

## 3. 회차 효율성 분석 (Turnaround Efficiency)
- **목표**: 종착역에 도착한 열차가 다시 반대 방향으로 투입되기까지의 소요 시간을 분석하여 회차 병목 구간을 확인합니다.
- **방법**:
    - 종착역에 도착(`train_status` 1)한 열차 번호가 다시 시점역에서 출발(`train_status` 0 또는 2)하는 시간 추적.
    - 회차 소요 시간 분포 분석.

## 4. 급행/일반 열차 간섭 분석 (Congestion/Overtake)
- **목표**: 급행 열차가 일반 열차를 추월하거나 앞선 열차 때문에 서행하는 구간을 탐지하여 운행 최적화 지점을 도출합니다.
- **방법**:
    - 급행(`is_express`=1)과 일반(`is_express`=0) 열차의 실시간 위치 관계 분석.
    - 선행 열차와의 거리가 좁혀지면서 속도(역간 이동 시간)가 줄어드는 구간 식별.
